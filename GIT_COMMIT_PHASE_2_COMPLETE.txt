Complete implementation of hybrid AirPlay 2 casting system with hardware H.264 
video encoding, Metal frame capture, and UDP/RTP network transmission.

PHASE 2a: H.264 VIDEO ENCODING
- VideoEncoder.mm: Hardware H.264 encoding via VideoToolbox
- Configurable bitrate (5-10 Mbps) and frame rate (60 FPS)
- Real-time encoding mode for <40ms latency
- Complete H.264 encoding chain with callbacks
- Frame statistics and latency tracking

PHASE 2b: METAL FRAME CAPTURE
- AirPlayFrameCapture.h/mm: Metal texture capture and conversion
- Integrated into GSDeviceMTL::EndPresent() rendering pipeline
- BGRA buffer extraction from GPU to CPU
- Synchronized timestamp tracking
- Frame capture lifecycle controlled by CastingManager

PHASE 2c: NETWORK TRANSMISSION
- AirPlayNetworkTransport.h/mm: UDP/RTP transport via Network Framework
- Connection management to AirPlay devices
- Non-blocking packet transmission with completion handlers
- Network statistics (packets sent/lost, bytes transmitted)
- Full error handling and recovery

ARCHITECTURE FLOW
Raw Game Frame (Metal)
    ↓
GSDeviceMTL::EndPresent() [Frame Capture]
    ↓
AirPlayFrameCapture::captureRenderTarget() [Metal→BGRA]
    ↓
AirPlayManager::submitVideoFrame() [Raw frame input]
    ↓
VideoEncoder::encodeFrame() [H.264 async encoding]
    ↓
AirPlayProtocol::encodeFrame() [RTP packaging]
    ↓
AirPlayNetworkTransport::sendRTPPacket() [UDP transmission]
    ↓
Apple TV/iPad (Receives RTP stream)
